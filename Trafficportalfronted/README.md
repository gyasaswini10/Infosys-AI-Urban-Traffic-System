## Role: Customer / Citizen (Traffic User, Not Booking-Centered)

### Core Purpose

Customer here means **city user who views traffic, plans travel, and gives traffic input**, not only booking vehicles.

### Functionalities

* Live Traffic Map

  * View real-time congestion levels
  * Color-coded roads (free, slow, blocked)
  * Accident, construction, event markers

* Smart Route Planner

  * Enter source & destination
  * AI suggests fastest, least congested, eco route
  * Compare 2–3 route options with ETA

* Traffic Alerts

  * Notifications for jams, accidents, diversions
  * Area-based alert subscription

* Traffic Search

  * Search traffic by location, time, incident type
  * History of past traffic patterns

* Community Traffic Reporting

  * Post traffic issues (accident, waterlogging, rally)
  * Upload photo + location
  * Verified by AI + admin

* Travel Time Predictor

  * Predict tomorrow’s traffic for same route
  * Peak hour warnings

* Public Transport View

  * See bus/metro delays due to traffic
  * Multimodal route planning

* Eco Travel Insights

  * CO₂ impact of chosen route
  * Suggest greener alternatives

## ⚙️ Setup & Installation

### 1. Backend (Spring Boot)
1.  Navigate to `TrafficPortal` folder.
2.  Update `application.properties` with your MySQL credentials.
3.  Run the application as a Spring Boot App.
4.  Server starts at `http://localhost:8080`.

### 2. Frontend (React)
1.  Navigate to `Trafficportalfronted` folder.
2.  Run `npm install` to install dependencies.
3.  Run `npm run dev` to start the development server.
4.  Open `http://localhost:5173` in your browser.

### 3. Login Credentials (Demo)
*   **Admin**: `admin@gmail.com` / `admin`
*   **Manager**: `manager@gmail.com` / `manager`
*   **Driver**: `driver@gmail.com` / `driver`
*   **Customer**: `customer@gmail.com` / `customer`
